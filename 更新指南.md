# 🔄 如何更新你的网页 (两种方法)

因为刚才你是直接拖拽上传的，你的电脑文件夹和 GitHub 还没有“连线”，目前最简单的方法是**直接覆盖**。

---

## 方法一：简单覆盖法 (推荐 ✅)
最适合像现在这样只改了几个文件的情况。

1.  打开你的 GitHub 仓库网页（就是刚才上传文件的那个页面）。
2.  点击右上角的 **Add file** 按钮。
3.  选择 **Upload files**。
4.  把你修改过的文件（这次主要是 `data.js` 和 `script.js`，或者干脆把整个文件夹里的文件再全选一次）**再次拖进去**。
5.  你会看到文件列表下面写着 "Update data.js" 之类的。
6.  直接点击绿色的 **Commit changes**。
7.  **搞定！** GitHub 会自动用新文件覆盖旧文件，过几分钟你的网页就更新了。

---

## 方法二：专业程序员法 (Git 命令行)
如果你想以后在 VS Code 里点一下就更新，需要执行以下步骤（只需要做一次设置）：
*(前提：你需要知道你的 GitHub 仓库地址，比如 `https://github.com/jacky/travel-diary.git`)*

1.  **初始化**：
    在 VS Code 的终端里输入：
    ```bash
    git init
    git add .
    git commit -m "第一版完整代码"
    git branch -M main
    ```

2.  **关联远程仓库**：
    你需要去 GitHub 页面，点击绿色的 **Code** 按钮，复制那个 HTTPS 地址，然后输入：
    ```bash
    git remote add origin https://github.com/你的用户名/你的仓库名.git
    ```

3.  **推送到 GitHub**：
    *(这一步可能会报错，因为 GitHub 上已经有文件了，需要先强制覆盖或者拉取，对新手比较复杂，所以建议用方法一)*
    ```bash
    git push -u origin main --force
    ```

---

### 💡 总结
目前建议你使用 **方法一**，简单无脑，把文件重新拖进去确认覆盖就行了！
